# Music-Hunter 🎵

**Music-Hunter** - это консольная утилита на Python, которая помогает:
- автоматически находить кэш **Яндекс.Музыки** на вашем компьютере;
- копировать скачанные треки в отдельную папку;
- переименовывать файлы по шаблону `Исполнитель - Название.mp3`, используя данные из API Яндекса.

Утилита ориентирована на Windows, но может работать и на других ОС (при наличии установленного клиента Яндекс.Музыки и небольших доработок путей).

---

## 📦 Возможности

- 🔍 **Автопоиск папки с кэшем** Яндекс.Музыки (в `%LOCALAPPDATA%\Packages\...`).
- 📋 **Копирование MP3‑файлов** в папку `music/` рядом с программой.
- 🏷️ **Переименование треков** через API Яндекса (требуется установленный клиент).
- ⚡ **Проверка обновлений** - автоматическая загрузка новой версии с GitHub.

---

## 🛠 Зависимости

Перед запуском установите необходимые библиотеки:

```bash
pip install yandex-music
```
Также программа использует встроенные модули:
- `pathlib`, `json`, `os`, `sys`, `shutil`, `zipfile`, `io`
- `requests` (должен быть установлен вместе с `yandex-music`).
> Если вы используете Windows, Python должен быть добавлен в `PATH` (или запускайте через `start.bat`).

---

## 🚀 Быстрый старт

1. **Клонируйте репозиторий** или скачайте последний релиз.
2. **Установите зависимости** (см. выше).
3. **Запустите программу**:
   - Windows: дважды щёлкните `start.bat` или выполните в терминале:
     ```bash
     python main.py
     ```
   - Linux/macOS:
     ```bash
     python3 main.py
     ```
4. При первом запуске программа найдёт папку с кэшем Яндекс.Музыки и сохранит её в `music_cache_path.json`.
5. Все MP3‑файлы будут скопированы в папку `music/` с читаемыми именами.

## 📂 Структура проекта

| Файл / Папка | Назначение |
|--------------|------------|
| `main.py` | Точка входа. Запускает проверку обновлений, поиск кэша, копирование и переименование. |
| `music_processor.py` | Логика копирования и переименования треков через API Яндекс.Музыки. |
| `cache_manager.py` | Поиск папки с кэшем, сохранение/загрузка пути. |
| `CheckingForUpdates.py` | Модуль автоматического обновления с GitHub Releases. |
| `banner.py` | Вывод цветного ASCII‑баннера в консоли. |
| `config.json` | Файл с текущей версией (используется при проверке обновлений). |
| `start.bat` | Пакетный файл для быстрого запуска на Windows. |
| `music/` | Папка назначения (создаётся автоматически). |

## 🔧 Требования

- Windows 10/11 (основная целевая ОС).
- Установленный и авторизованный клиент Яндекс.Музыки (для получения корректных названий треков).
- Python 3.8+.
- Библиотека `yandex-music`.

---

## 📝 Примечания

- Если клиент Яндекс.Музыки не инициализируется (например, нет активной сессии), файлы копируются без переименования — с исходными числовыми именами.
- Программа **не скачивает** музыку из интернета, а лишь обрабатывает уже загруженные через официальное приложение файлы.
- При обновлении все файлы проекта (включая `.bat`) заменяются новыми из архива релиза.
